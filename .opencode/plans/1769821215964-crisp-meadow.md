# UI Redesign Plan: Modern Speed Reader

## Overview
Transform the speed reading PWA into a polished, modern experience with:
- **Design Philosophy**: "Focused Elegance" - A refined dark interface that feels premium and distraction-free, using subtle depth and purposeful motion
- **Dark Theme**: Enhanced with better contrast, refined accent colors, and improved visual hierarchy
- **Animations**: Smooth transitions, micro-interactions, and reading-focused motion design

---

## 1. Design System Updates

### Color Palette Refinement
**Current:** Slate-950 background with emerald accents
**Proposed:**
- **Background**: Layered dark tones (`bg-neutral-950` to `bg-neutral-900`) for depth
- **Surface**: Subtle gradients (`from-neutral-900/90 to-neutral-800/90`)
- **Primary Accent**: Violet-500 (better for reading apps - less eye strain than emerald)
- **Secondary Accent**: Cyan-400 for highlights and interactive states
- **Text Hierarchy**:
  - Primary: `text-neutral-100` (headings, important text)
  - Secondary: `text-neutral-300` (body text)
  - Tertiary: `text-neutral-500` (metadata, hints)
- **Semantic Colors**:
  - Success: Emerald-400 (keep for progress)
  - Highlight: Amber-300/400 (for speed reading focus word)

### Typography System
**Current:** System fonts only
**Proposed:**
- **Reading Font**: Inter or Geist (clean, excellent readability)
- **Display Font**: Same family, heavier weights for headers
- **Font Sizes**: More granular scale with better line-height
- **Reading Mode**: Slightly increased line-height (leading-loose) for comfort

### Spacing & Layout
- Increased padding for touch targets (min 44px)
- Consistent 4px/8px spacing scale
- Better breathing room between elements
- Refined border-radius (use `rounded-2xl` and `rounded-3xl` consistently)

---

## 2. Animation Strategy

### Page Transitions
**Implementation**: Use Next.js App Router with CSS transitions
- **Library → Reader**: Fade + slight scale (0.98 → 1) + slide up (20px)
- **Mode Switching** (Speed ↔ Normal): Crossfade with 300ms duration
- **Duration**: 300-400ms with `ease-out` timing

### Micro-interactions
**Buttons & Cards**:
- Hover: Subtle lift (`translateY(-2px)`) + shadow increase
- Active: Scale down (0.97) with spring-like feel
- Focus: Ring animation with violet accent
- Duration: 150-200ms

**Progress Bar**:
- Smooth width transitions (300ms ease-out)
- Glow pulse on completion

**Reading Controls**:
- Speed reading controls: Slide up/down with backdrop fade
- Settings modal: Scale + fade from center
- Chapter menu: Slide from top with staggered list items

### Reading Mode Animations
**Speed Reading**:
- Word transitions: Subtle fade (50ms) to reduce jarring
- Play/pause button: Pulse animation when active
- WPM indicator: Number tick animation on change

**Normal Reading**:
- Paragraph fade-in on virtual scroll (staggered)
- Highlighted word: Gentle glow pulse (not static background)
- Chapter change: Smooth scroll with progress bar update

### Technical Implementation
**Libraries to use**:
- Native CSS transitions (preferred for performance)
- `framer-motion` for complex sequences (if needed)
- CSS `@keyframes` for repeating animations (pulse, glow)

**Performance Rules**:
- Use `transform` and `opacity` only (GPU accelerated)
- Respect `prefers-reduced-motion` media query
- Keep animations under 300ms for responsiveness

## 4. Global Style Updates

### globals.css Enhancements
```css
/* Smooth scrolling */
html {
  scroll-behavior: smooth;
}

/* Selection color */
::selection {
  background: rgba(139, 92, 246, 0.3);
  color: inherit;
}

/* Reduced motion support */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

/* Custom scrollbar for webkit */
::-webkit-scrollbar {
  width: 8px;
  height: 8px;
}
```

## 9. Verification Steps

1. **Visual Check**: Open app in browser, verify all components render correctly
2. **Animation Check**: 
   - Test page transitions (Library ↔ Reader)
   - Test mode switching (Speed ↔ Normal)
   - Test all hover/active states
3. **Mobile Check**: Test on actual mobile device or DevTools mobile view
4. **Accessibility Check**: 
   - Enable `prefers-reduced-motion` in OS settings, verify animations disable
   - Check contrast ratios meet WCAG AA
5. **Performance Check**: 
   - Open DevTools Performance tab
   - Record while using app, verify no long tasks or layout shifts
